## Repos to Create

Think of each repo as a portfolio piece and a talking point in interviews:

### resume

— Your resume in Markdown + exported PDF. Version it. Employers almost never see a versioned resume and it signals discipline.

### portfolio-projects

— A curated showcase repo with a well-written README linking to your best work, even if the actual code is private or at a past employer (describe the project, your role, the stack, the outcome).

### job-search-tracker

— Your Kanban/CRM (GitHub Project board tied to this repo). Use Issues for each company, labels for status.

### interview-prep

— Notes, coding challenge solutions, system design diagrams. Even if kept private, having it in GitHub keeps you organized. You could make select parts public to show thoroughness.

### til (Today I Learned)

— Short daily or weekly entries of things you're learning. This is a well-known pattern in the dev community, shows continuous growth, and gives recruiters something to follow.

## GitHub Projects as a Kanban board

Use GitHub Projects as a Kanban board — Track applications with columns like Researching → Applied → Phone Screen → Interview → Offer → Closed. This is genuinely useful and also shows process thinking if anyone sees it.

## Use GitHub Issues as a CRM

Log each company as an Issue with labels (recruiter-reach-out, applied, referral, etc.) and due dates for follow-ups. Lightweight but powerful.

If you're also active on LinkedIn, consider aligning your headline or URL slug to reinforce the same professional identity.
